# DarkLab Research Protocol

Standardized methodology for running autonomous research through the DarkLab agent swarm. Every research task follows this protocol to ensure reproducibility, quality, and proper archiving.

## 1. Research Modes

| Mode | Command | Agents | Time | Best For |
|------|---------|--------|------|----------|
| **Quick** | `/deerflow <topic>` | 1 (DeerFlow) | 30-60s | Single-question answers, fact lookup |
| **Deep** | `/deepresearch <topic>` | 1 (Codex+Memento loop) | 5-15 min | Comprehensive literature review |
| **Swarm** | `/swarmresearch <topic>` | 3-5 parallel | 15-30 min | Multi-angle analysis, novel pathways |
| **Debate** | `/debate <topic>` | 2+ adversarial | 20-40 min | Competing hypotheses, solution comparison |
| **Scheduled** | `/schedule <topic>` | 1 (recurring) | 5-15 min/run | Monitoring, trend tracking |

## 2. Research Pipeline

```
PHASE 1: PLANNING
  Input: research topic (natural language)
    |
    v
  Query Decomposition (LLM breaks topic into 3-5 sub-queries)
    |
    v
  Mode Selection (quick/deep/swarm/debate based on complexity)
    |
    v
  Prior Knowledge Check (search knowledge base for related past research)

PHASE 2: SEARCH
  Sub-queries → 9 Academic Sources (parallel)
    |
    ├── arXiv (CS, physics, math, bio preprints)
    ├── Semantic Scholar (200M+ papers, citation graph)
    ├── PubMed (36M+ biomedical citations)
    ├── OpenAlex (250M+ works, open metadata)
    ├── CrossRef (150M+ DOI records)
    ├── CORE (300M+ open access papers)
    ├── Europe PMC (PubMed + PMC + preprints)
    ├── DOAJ (10M+ open access journal articles)
    └── DuckDuckGo (web fallback for non-academic sources)
    |
    v
  Deduplication (normalize titles, track seen papers globally)

PHASE 3: SYNTHESIS (Codex-style modify-verify-decide)
  Papers + Context → LLM Synthesis
    |
    v
  Structured Output:
    ## Key Findings (with [citations])
    ## Methods and Approaches
    ## Current Impact
    ## Open Questions and Future Directions
    ## References

PHASE 4: REFINEMENT (Memento-style skill swarm)
  Draft → Fact-Checker → Gap-Finder → Synthesizer
    |
    v
  Refined Output (gaps filled, errors corrected, structure improved)

PHASE 5: EVALUATION
  Refined Output → Quality Scorer (5 metrics)
    |
    ├── Completeness (0-1): all aspects covered?
    ├── Source Quality (0-1): peer-reviewed citations?
    ├── Structure (0-1): clear sections, logical flow?
    ├── Novelty (0-1): non-trivial insights?
    └── Accuracy (0-1): facts correct, properly attributed?
    |
    v
  Weighted Score (threshold: 0.75)
    |
    ├── Score ≥ 0.75 → DONE (archive + report)
    └── Score < 0.75 → ITERATE (gap-directed search, max 5 rounds)

PHASE 6: ARCHIVAL
  Result → results/research/<date>_<topic_slug>.md
  Papers → results/research/<date>_<topic_slug>_sources.json
  Lessons → ~/.darklab/memento-codex/global_lessons.jsonl
  Knowledge → ~/.darklab/memento-codex/knowledge.jsonl
  Telegram → structured report to user
```

## 3. Swarm Research Protocol

When running multi-agent research (`/swarmresearch`):

### Agent Assignment
```
Topic Analysis → Decompose into N research angles
  |
  ├── Agent A: Foundations (classical theory, established methods)
  ├── Agent B: State of the Art (recent advances, current SOTA)
  ├── Agent C: Novel Pathways (unconventional approaches, cross-domain)
  ├── Agent D: Computational (simulation, verification, AI methods)
  └── Agent E: Practical (applications, implementation, feasibility)
```

### Execution
1. All agents search **independently** (different sub-queries per angle)
2. Each produces a **scored draft** (Phase 3-5)
3. Results are **merged** with cross-reference deduplication
4. A **synthesis agent** combines all findings into a unified report
5. Final report includes **per-agent attribution** for traceability

### Debate Protocol
For `/debate` mode, agents take **adversarial positions**:
```
Agent PRO: argues FOR the hypothesis/approach
Agent CON: argues AGAINST, identifies weaknesses
Agent JUDGE: evaluates both arguments, scores evidence quality
  |
  v
Synthesis: balanced analysis with confidence levels per claim
```

## 5. Quality Gates

| Gate | Threshold | Action if Failed |
|------|-----------|-----------------|
| Minimum papers | ≥ 5 | Broaden search queries |
| Source diversity | ≥ 3 databases | Add missing source queries |
| Citation coverage | ≥ 80% claims cited | Fact-checker pass |
| Structure score | ≥ 0.7 | Reformat with enforced sections |
| Overall score | ≥ 0.75 | Iterate (max 5 rounds) |
| Max iterations | 5 | Deliver partial with caveat |

## 7. Knowledge Base Integration

Every completed research feeds back into the system:

1. **Knowledge JSONL** (`~/.darklab/memento-codex/knowledge.jsonl`)
   - Topic, score, summary, paper count, timestamp
   - Searched by keyword overlap for future requests
   - Enables: "building on 2 prior research entries"

2. **Global Lessons** (`~/.darklab/memento-codex/global_lessons.jsonl`)
   - What worked, what didn't, strategy effectiveness
   - Applied to future research via prompt injection
   - Enables: continuous improvement across requests

3. **Scheduled Monitoring** (`~/.darklab/memento-codex/schedule.json`)
   - Auto-re-research topics on interval (24h, 48h, weekly)
   - Detects new papers and updates knowledge base
   - Reports changes via Telegram
